# v0.0.1-rc22

Post-tool prose reuse enabled by default.

## Summary

- Enable `ORBIT_POST_TOOL_FINAL_REUSE` by default for eligible native tool-loop completions.
- Keep `ORBIT_POST_TOOL_FINAL_REUSE=0` as the immediate kill switch.
- Reuse the model's existing post-tool prose without changing routing, tool selection, execution, canonical validation, healing, MTP, or final-prefix reuse.

## Behavior

After a terminal `post_tool_route` result, Orbit reuses the exact byte-for-byte `ChatResult.content` only when all structural eligibility checks pass:

- the result stopped normally with `finish_reason=stop`;
- a terminal tool result is complete;
- no new tool call, retry, pending guardrail, error, or additional step exists;
- the prose is non-empty and contains no tool markup, control markup, or technical JSON;
- the existing completeness checks accept the response.

Any uncertainty falls back automatically to the normal `final_from_tool` path. Runtime does not summarize, construct, classify semantically, or modify the response. No retry is added.

## Configuration

- Unset `ORBIT_POST_TOOL_FINAL_REUSE`: enabled, source `default`.
- `ORBIT_POST_TOOL_FINAL_REUSE=1`: enabled, source `stable`.
- `ORBIT_POST_TOOL_FINAL_REUSE=0`: disabled, source `stable`.
- Invalid values: disabled safely with a bounded configuration error.

Diagnostics expose bounded effective state, source, reuse/fallback counters, and fallback reason. Raw prompts, model output, evidence, arguments, paths, URLs, and credentials are not recorded.

## Results

Process-isolated Gemma 4 12B validation recorded 50/50 comparable OFF cases and 50/50 correct, stopped ON reuses:

- 50 model calls eliminated;
- 65,189 evaluated tokens saved;
- median saving of 1,084 evaluated tokens per reuse;
- approximately 107.5 seconds median wall-time saving per reuse in the measured workload;
- zero false positives and zero skipped tools.

The response, tool, arguments, finish reason, lifecycle, and correctness remained unchanged in the validated cases. The eligibility-analysis p95 was approximately 8.55 microseconds.

## Limitations

- Reuse applies only to structurally eligible post-tool prose. Read, system, web, error, synthesis, multi-tool, cancellation, timeout, and reset cases fall back when eligibility is not proven.
- This release makes no deterministic speedup claim. CPU timing depends on workload, output length, process state, and thermal conditions.
- The feature does not correct semantic tool selection, missing tools, incomplete model output, or other model-adherence failures.
