## Experimental final-prefix reuse

Set `ORBIT_FINAL_PREFIX_EXPERIMENT=1` to enable the experiment. It remains OFF
by default and is limited to eligible native `final_from_tool` calls with tools
enabled.

Eligibility requires the exact prompt family, role sequence, and validated
43-token prefix. The first eligible call captures the checkpoint; later
eligible calls can restore it. A prefix mismatch or restore failure falls back
to normal prefill. Cancel, reset, and completion errors invalidate checkpoint
state.

The experiment is ineligible when MTP owns the completion path. It does not
read, mutate, reset, or invalidate MTP state. Bounded `/props` diagnostics expose
enabled and initialized state, prefix tokens, capture/restore/fallback counters,
failure reason, last-use state, and checkpoint size.

## Token and cache result

- Default behavior remains `cached=4` on route-to-final.
- Eligible experimental restored calls reach `cached=43`.
- The exact net reduction relative to default behavior is 39 evaluated tokens.
- The first eligible call captures; subsequent eligible calls restore the validated 43-token prefix.
- `cached=4` is unchanged and unresolved when the experiment is disabled.

## Benchmark harness

The smoke harness can manage native-server startup and propagate the requested
OFF/ON mode to both server and runtime client, without a manual environment
export. It records capture, restore, and fallback counters; cached and evaluated
tokens; output length; route, final, non-model, and total timing; run and block
order; CPU affinity; managed PID; and bounded `/props` snapshots.

Repeatable coverage includes deterministic web success, no-result, and error
fixtures; tools-off validation; server restart; context change; thinking-mode
eligibility; cancel and timeout cleanup; MTP guard behavior; and ordered RSS/PID
sampling through capture, restores, invalidation, and recapture. Existing JSONL
environment and step rows remain compatible; new summaries are additive.

## Performance interpretation

The 39-token evaluated reduction on eligible restored calls is deterministic.
Controlled matched-output smoke cases showed lower final latency, and measured
checkpoint restore overhead was only a few milliseconds.

These observations do not establish a deterministic wall-time improvement.
Output length, process order, workload, and thermal state affect CPU timing; no
general speedup percentage should be advertised.

## Known limitations

- The experiment remains OFF by default and opt-in only.
- Experimental logits differ from cold full-prefill because segmentation changes.
- Restore is bit-exact against an identically segmented baseline.
- Non-stream HTTP timeout may require explicit `/cancel` for reliable cleanup.
- RSS shows bounded allocator and page-touch variation; no repeated linear-growth pattern was observed, but this is not a general no-leak guarantee.
- MTP and final-prefix reuse are mutually exclusive by design.
- This release does not solve default `cached=4` behavior or reopen route-to-final cache redesign.
